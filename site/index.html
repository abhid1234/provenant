An agent wrote this file.
Prove which one, and why.

Coding agents now write a large share of a repo — but the work lands with no verifiable record of where it came from. git blame shows the human who ran the agent, not the model, the intent, or the artifacts it derived from. provenant writes a small, notarized attestation for each artifact — a receipt whose id is the sha256 of its own content and whose artifact is the sha256 of the file's bytes — into an append-only, content-addressed ledger anyone can verify offline.

Every tab runs the actual published provenant code, imported as ES modules — the same functions npm i @avee1234/provenant gives you. Idempotent by construction. The record's id is the sha256 of its own canonical content, so attesting the same artifact + intent twice yields the same id — a duplicate append is a no-op the ledger fold collapses.

The real chainOf(id, ledger) follows each record's parents[] back through its derivation ancestry — depth-first, deduped, and cycle-safe — from the newest seal all the way to origin. Pick an attestation and walk it.

Best-effort over what's present. A parent id with no matching record is skipped rather than invented, and a diamond or cycle is visited once — so the walk is always a finite, honest ancestry, never a fabricated one.

Re-hash an artifact's current bytes and fold the ledger with the real verify(hash, ledger) — ATTESTED (by whom, when, and under what intent), UNKNOWN, or REVOKED. Verification never needs the bytes stored in the ledger; the digest alone decides.

The real coverage(hashes, ledger) over every artifact in the working set — the fraction that carries a live attestation, with a per-file breakdown re-run through verify(). This is the repo audit: how much of the agent's output can prove where it came from.

The two integrity demos. Tamper flips one byte of an artifact — its fingerprint stops matching any record, so verify returns UNKNOWN. Revoke leaves the bytes alone but appends a record that supersedes the attestation, flipping the seal to REVOKED. Both are caught offline, from the ledger alone.

Map attestations to OpenTelemetry span attributes — attach to any span (Arize / LangSmith / Braintrust…). Deterministic and zero-dependency. No clock, no crypto, no filesystem — a record in, a flat attribute object out. Arrays are joined to comma strings and revocation context appears only on a revoked record, so every emitted value is a bare scalar an exporter can ship as-is.

Cross-org tamper-evidence with ed25519. The author signs a record with a private key nobody else holds; anyone verifies with the matching public key — no shared secret. Append-only · content-addressed

One JSON record per line — every tab reads and writes this same log. Each id is a content hash, so duplicate appends are idempotent and a tampered line no longer matches its id, dropped on read by the real resolveRecords() fold.

Quickstart — adopt it in a minute

Three ways to wire it in.

A CLI for one-off attestation and audit, a zero-dependency library for your own agent runtime, or a hook that auto-attests every file a commit changes.

① CLI & audit

Attest what you produced, then verify or audit coverage across the repo — offline, no install.

# no install needed
npx @avee1234/provenant \
  attest src/auth.js \
  --intent "add OAuth" --agent claude
npx @avee1234/provenant verify src/auth.js
npx @avee1234/provenant coverage 'src/**/*.js'

② Library

Attest and verify inside your own agent loop. Pure ESM, zero dependencies, fully deterministic.

import { attest, verify, loadLedger }
  from '@avee1234/provenant';

// record what the agent produced
const rec = await attest(bytes, {
  agent, intent, created });

③ Auto-attest hook

A drop-in git / Claude Code hook: attest every file a commit changes. Self-applying provenance.

# install the pre-commit hook
npx @avee1234/provenant hook install

# every commit now appends
# an attestation per changed file
How it's built — no magic

The playground runs this exact source.

provenant is a few hundred lines of pure, dependency-free JavaScript. The playground imports these same files as browser ES modules — verify.js and schema.js are copied verbatim from src/; id.js ports the record constructors and the ledger fold, swapping only Node's sha256 for the browser's crypto.subtle. What you poke at above is the library. Expand any file to read the real implementation, fetched live from /lib.

What's in the box

Five verbs, one hook, an open format.

provenant attest

Attest

Record which agent produced which artifact, why, and derived from what — a content-hash-id'd receipt.

provenant verify

Verify

Re-hash a file and fold the ledger offline: ATTESTED by whom, UNKNOWN, or REVOKED.

provenant chain

Chain

Walk an attestation's parents[] back through its derivation ancestry to origin — cycle-safe.

provenant coverage

Coverage

Audit a repo: what fraction of its artifacts carry a live attestation, and exactly which don't.

provenant revoke

Revoke

Supersede an attestation with an appended record — the ledger stays append-only, never a deletion.

Meta

Built — and attested — by an autonomous agent factory.

Every feature of provenant was triaged, specced, implemented, adversarially reviewed, and released by an autonomous "software factory" of agents, with a human approving the gates. The tool that proves which agent produced which artifact was itself produced by agents — and now runs on the very sessions that built it. The ledger it writes is the trail it was designed to leave.
