Which agent, under what intent, derived from what? provenant writes a small, verifiable attestation for each artifact — a notarized receipt whose id is the sha256 of its own content and whose artifact is the sha256 of the file's bytes. 06 revoke

Supersede an attestation

A revocation is appended, never a deletion — the ledger stays append-only and verify folds it in at read time. Watch a seal flip to REVOKED.

Tamper vs. revoke. Tampering changes an artifact's bytes — its fingerprint stops matching any record, so verify returns UNKNOWN. Revoking leaves the bytes alone but appends a record that supersedes the attestation. Both are caught offline, from the ledger alone.

05 the ledger · .provenant/ledger.jsonl

Append-only · content-addressed

One JSON record per line. Every id is a content hash, so duplicate appends are idempotent and a tampered line no longer matches its id — dropped on read by the real resolveRecords() fold.
